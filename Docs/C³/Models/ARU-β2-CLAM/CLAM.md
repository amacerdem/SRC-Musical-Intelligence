## 1. What Does This Model Simulate?

The **Closed-Loop Affective Modulation** (CLAM) model describes how a bidirectional brain-music interface can read affective states from neural signals and generate music to modulate them in real-time, forming a causal feedback loop.


### 1.1 Why This Matters for ARU

CLAM provides the **closed-loop control complement** to SRP's open-loop reward model:
- **SRP** answers: "What is the reward response?" (measurement, open-loop)
- **CLAM** answers: "Can we steer the reward response?" (intervention, closed-loop)
- CLAM's error signal uses SRP's reward/valence as the decoded state
- CLAM's modulation success validates SRP's causal role
- Together, SRP + CLAM demonstrate music-affect causality (not just correlation)

### 3.2 The P-Control Law (Ehrlich 2019)

```
CLOSED-LOOP CONTROL MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━

u(t) = K_p × e(t)

where:
  u(t)   = control output (music parameter adjustment)
  K_p    = proportional gain (~0.5-2.0, tuned per individual)
  e(t)   = target_state - current_state (error signal)

Music parameter mapping:
  tempo_delta(t)  = u_arousal(t) × tempo_sensitivity   (≈20 BPM/unit)
  mode_delta(t)   = u_valence(t) × mode_sensitivity    (≈0.5 major-minor)

Closed-loop transfer function:
  H(s) = Kp·G_music(s)·G_brain(s) / (1 + Kp·G_music(s)·G_brain(s))

Total loop latency: ~1000ms
  t=0ms:    EEG acquisition (FC6)
  t=100ms:  Feature extraction (gamma power)
  t=200ms:  Affective state decode
  t=400ms:  Control law + music parameter update
  t=600ms:  Audio generation
  t=1000ms: Brain response (loop closes)
```

## 7. Mechanism Computation

### 7.1 AED Binding (Primary: Affect State Tracking)

```python
# AED reads: arousal_dynamics[0:10], expectancy_affect[10:20]
# Horizons: H6(200ms) and H16(1000ms), bidirectional

# Current decoded affective state from AED
affect_fast = AED.arousal_dynamics[0:4].mean()    # H6: 200ms instant affect
affect_slow = AED.expectancy_affect[10:14].mean()  # H16: 1s integrated affect

# Decoded state x(t) — slow dominates for stability
decoded_affect = tanh(0.3 * affect_fast + 0.7 * affect_slow)
# Coefficients: |0.3| + |0.7| = 1.0 (no saturation)
# Slow integration dominates — BCI needs stable state estimate

# State confidence from entropy
state_entropy = AED.arousal_dynamics[6]  # M20(entropy) at H16
state_confidence = 1.0 - state_entropy   # Low entropy = confident decode
```

### 7.2 CPD Binding (Secondary: Modulation Peak Detection)

```python
# CPD reads: trigger_features[0:10], buildup_tracking[10:20]
# Horizons: H7(250ms), H12(525ms), bidirectional

# Modulation success detection via CPD
success_instant = CPD.trigger_features[0:4].mean()    # H7: 250ms instant peak
success_trend = CPD.buildup_tracking[14:18].mean()     # H12: 525ms trend

# Loop coherence — Granger causality proxy
coherence_fast = CPD.trigger_features[4:8].mean()      # H7: 250ms coherence
coherence_trend = CPD.buildup_tracking[10:14].mean()   # H12: 525ms coherence

# Combined loop coherence
loop_coherence = sigmoid(0.4 * coherence_fast + 0.6 * coherence_trend)
# Coefficients: |0.4| + |0.6| = 1.0

# Modulation trend (is loop converging?)
modulation_trend = tanh(success_trend * 2.0)
```

### 7.3 Control Law (No C0P — control computed from AED + CPD)

```python
# P-control law from decoded state and target
# Kp adapted by state confidence

K_p_base = 1.0
K_p = K_p_base * (0.5 + 0.5 * state_confidence)
# High confidence → full gain, low confidence → half gain

# Error signal
error = target_affect - decoded_affect

# Control output
control = tanh(K_p * error)

# Music parameter delta
music_delta = control * sigmoid(loop_coherence * 3.0)
# Modulation only when loop is coherent
```

### 7.4 CLAM Output Computation

```python
def compute_clam(R3, H3, AED, CPD, target_affect):
    """
    CLAM: 11D output per frame.

    All deterministic. Zero learned parameters.
    Coefficients from Ehrlich 2019.
    """
    # --- Decoded State ---
    decoded = compute_decoded_affect(AED, R3)   # x(t) [-1,1]
    confidence = compute_confidence(AED)         # [0,1]

    # --- Loop Coherence ---
    coherence = compute_loop_coherence(CPD)      # [0,1]

    # --- Error and Control ---
    error = target_affect - decoded
    K_p = 1.0 * (0.5 + 0.5 * confidence)
    control = tanh(K_p * error)
    music_delta = control * sigmoid(coherence * 3.0)

    # --- Layer E: Explicit Features ---
    f09 = 1.0 - abs(error)                         # Modulation success
    f09 = clamp(f09, 0, 1)
    loop_coh = coherence                             # Loop sync quality

    # --- Layer B: BCI State ---
    dec_affect = decoded                             # Current state
    tgt_affect = target_affect                       # Target state
    err_signal = error                               # Error

    # --- Layer C: Control ---
    ctrl_out = control                               # P-control output
    music_param = music_delta                        # Music adjustment

    # --- Layer P: Present Processing ---
    arousal_mod = sigmoid(
        0.5 * AED.arousal_dynamics[0:4].mean() +
        0.5 * CPD.trigger_features[0:4].mean()
    )  # |0.5| + |0.5| = 1.0
    valence_track = tanh(
        decoded * coherence
    )  # Valence quality scaled by loop coherence

    # --- Layer F: Future Predictions ---
    # Error trend from CPD buildup → convergence prediction
    error_trend = CPD.buildup_tracking[14:18].mean()
    target_pred = tanh(decoded + 0.5 * error_trend)
    success_pred = sigmoid(coherence + 0.5 * (1.0 - abs(error)))

    return stack([
        f09, loop_coh,                            # E: 2D
        dec_affect, tgt_affect, err_signal,        # B: 3D
        ctrl_out, music_param,                     # C: 2D
        arousal_mod, valence_track,                # P: 2D
        target_pred, success_pred                  # F: 2D
    ])  # Total: 11D
```
