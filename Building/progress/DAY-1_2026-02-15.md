## 2. BCH: First End-to-End Nucleus

### The Model

**BCH** (Brainstem Consonance Hierarchy) implements the auditory brainstem's consonance processing circuit based on Bidelman & Krishnan (2009). It is an SPU (Sensory Processing Unit) Relay — depth 0, no mechanisms, reads raw R3 and H3 features directly.

| Property | Value |
|----------|-------|
| Unit | SPU (Sensory Processing Unit) |
| Tier | alpha-1 (Relay) |
| Output | 12D: E(4) + M(2) + P(3) + F(3) |
| Key R3 inputs | roughness[0], sethares[1], helmholtz[2], stumpf[3], tonalness[14], autocorr[15] |
| Brain regions | CN, SOC, IC, MGB, A1_HG (ascending auditory pathway) |
| Neurochemicals | DA (reward for consonance), NE (arousal for dissonance) |
| Code | `Musical_Intelligence/brain/units/spu/relays/bch.py` — 700 lines |
| Doc | `Docs/C3/Models/SPU-a1-BCH/BCH.md` — 777 lines |

### Key Formulas (from code)

```
E-layer (Extraction):
  f01_nps      = sigmoid(ALPHA * tonalness * autocorr)
  f02_harm     = sigmoid(ALPHA * helmholtz * stumpf)
  f03_hier     = sigmoid(BETA * (1 - roughness) * (1 - sethares))
  f04_ffr      = sigmoid(FFR_CORR * (tonalness + autocorr) / 2)

M-layer (Mechanism):
  f05_binding   = (f01 + f02 + f03 + f04) / 4
  f06_stability = sigmoid(f05 * GAMMA)

P-layer (Cognitive):
  consonance_signal = 1 - (roughness + sethares) / 2
  template_match    = (helmholtz + stumpf) / 2
  neural_pitch      = f01_nps

F-layer (Forecast):
  interval_expect  = sigmoid(BETA * f05_binding * consonance_signal)
  pitch_track      = sigmoid(GAMMA * neural_pitch * template_match)
  resolution_pred  = sigmoid(GAMMA * consonance_signal * f06_stability)
```

## 3. R3 Consonance Upgrade

### The Problem

ConsonanceGroup [0:7] used mel-spectrogram proxy computations that lacked frequency resolution for psychoacoustic analysis:

| Feature | Old (Mel-Proxy) | Result |
|---------|-----------------|--------|
| roughness [0] | `sigmoid(mel_high.var / mel.mean)` | **Constant** across all intervals (0.3775) |
| sethares [1] | `diff(mel, bins).abs().mean()` | Inverted — Unison showed as most dissonant |
| helmholtz [2] | lag-1 autocorrelation of mel bins | Weak proxy, ~0.5% spread |

### The Fix

Backward-compatible audio passthrough: `R3Extractor.extract(mel, *, audio=None, sr=44100)`

When `audio` is provided, ConsonanceGroup uses real psychoacoustic models:

| Model | Paper | Implementation |
|-------|-------|---------------|
| **Sethares dissonance** | Sethares 1993, 2nd ed. | Pairwise dissonance: D*=0.24, A1=-3.51, A2=-5.75; STFT peak-picking (n_fft=4096), top-20 partials |
| **Plomp-Levelt roughness** | Plomp & Levelt 1965 | Zwicker critical bandwidth weighting on Sethares pairs |
| **HPS F0 estimation** | Harmonic Product Spectrum | R=5 downsample orders on magnitude spectrum |
| **Harmonic template matching** | Terhardt, Stoll & Seewann 1982 | Classify partials as harmonic (within 5% of n*F0), compute E_harmonic/E_total |

### Files Modified (4 core + 2 callers)

| File | Change | Lines |
|------|--------|-------|
| `contracts/bases/base_spectral_group.py` | Added `compute_from_audio()` default method | +15 |
| `ear/r3/pipeline/stage_executor.py` | Try audio path first, fallback to mel | +20 |
| `ear/r3/extractor.py` | Pass `audio`, `sr` through to executor | +10 |
| `ear/r3/groups/a_consonance/group.py` | Complete rewrite with real models | +488 (41 -> 529) |
| `Lab/WebLab/scripts/run_pipeline.py` | Pass waveform tensor to extract() | +3 |
| `Musical_Intelligence/hybrid/calibration.py` | Pass audio tensor to extract() | +3 |

Synthetic piano dyads (C4 base, 6 canonical intervals):

## 7. Composer Evaluation

The final validation was not computational but musical. The composer:

1. Loaded the `beethoven-pathetique` experiment (Beethoven Sonata No. 8, Op. 13, Adagio cantabile)
2. Played 36 seconds on piano, reading from notation
3. Compared the felt consonance at each moment with the BCH consonance signal in real time
4. Checked that dissonant passages (diminished chords, chromatic motion) showed lower consonance values
5. Checked that consonant passages (tonic arrivals, perfect cadences) showed higher consonance values

**Score: 96/100**

The 4-point deduction likely comes from edge cases at phrase boundaries and rapid harmonic transitions — areas where the current frame-level analysis (172 Hz) may smooth over micro-timing nuances that a performer perceives in the act of playing.
