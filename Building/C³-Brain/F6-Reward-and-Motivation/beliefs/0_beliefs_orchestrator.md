## 2. Belief Categories

### Core (C) -- Full Bayesian PE Cycle

Core beliefs undergo predict -> observe -> update every frame:

```
predicted = tau * prev + (1-tau) * baseline + trend + periodicity + context
observed  = f(mechanism outputs)
gain      = pi_obs / (pi_obs + pi_pred + eps)
posterior = (1 - gain) * predicted + gain * observed
PE        = observed - predicted  ->  feeds reward formula
```

F6 Core beliefs are **terminal**: they do not observe sensory data directly. Instead, F6 aggregates prediction errors from ALL 36 Core Beliefs across 8 Functions (F1-F5, F7-F9) via the RewardAggregator. F6.prediction_error is the reward PE (TD error), distinct from per-belief sensory PEs.

F6 Core beliefs operate at **single-scale** in v1.0 kernel (multi-scale extension deferred).

SRP Core beliefs:
- **`wanting`** (tau=0.6 -- incentive salience ramps quasi-hyperbolically before reward events)
- **`liking`** (tau=0.65 -- hedonic impact peaks at consummation, slightly slower decay)
- **`pleasure`** (tau=0.7 -- aggregate hedonic state, highest inertia reflects slow pleasure evolution)
- **`prediction_error`** (tau=0.5 -- fastest F6 core, TD error responds rapidly to surprise)
- **`tension`** (tau=0.55 -- anticipatory arousal, builds before events, faster than pleasure)

### Appraisal (A) -- Observe Only

Appraisal beliefs are computed directly by mechanisms. No predict/update cycle, no PE, no tau. They are evaluative judgments tracked in BeliefStore.

SRP Appraisals: **`prediction_match`**, **`peak_detection`**, **`harmonic_tension`**
DAED Appraisals: **`dissociation_index`**, **`temporal_phase`**, **`da_caudate`**, **`da_nacc`**

### Anticipation (N) -- Forward Predictions

Anticipation beliefs are forward predictions generated by F-layer outputs. They feed Core beliefs' predict() method and the precision engine.

SRP Anticipation: **`chills_proximity`**, **`resolution_expectation`**, **`reward_forecast`**
DAED Anticipation: **`wanting_ramp`**

### Key Principle: Mechanism != Belief

- **Mechanism** (mechanism/ folder): HOW the computation works (R3 inputs, H3 integration, weighted sums)
- **Belief** (this folder): WHAT the listener cognitively infers ("I want more", "that felt good")

## 4. Phase Schedule

F6 models operate in **Phase 5** (terminal -- after all other functions):

```
Phase 0a: BCH || HMCE || SNEM || MEAMN || MPG || SRP || PEOM  (7 independent relays)
Phase 0b: DAED (cross: BCH+MEAMN), HMCE (cross: reads other relays)
Phase 0c: Consonance multi-scale (BCH->HTP), Tempo multi-scale (PEOM->HMCE)
Phase 1:  F2 prediction + F3 attention (parallel)
Phase 2:  F4 memory + F5 emotion (parallel)
Phase 2a: Predict/Observe for all Core beliefs
Phase 2b: PE + Precision
Phase 2c: Update (Bayesian posterior)
Phase 3:  Reward aggregation
  |-- F6: SRP + DAED mechanism outputs available from Phase 0a/0b
  |        |-- SRP reads: salience signal + all PEs from F1-F5 + F7-F9
  |        |-- DAED reads: BCH consonance + MEAMN memory + SRP reward signals
  |-- F6 Core beliefs: predict/observe/update cycle
Phase 5:  F6 terminal (wanting, liking, pleasure, tension, prediction_error)
```

**F6 is terminal**: it consumes PEs from all other Functions but does not feed forward to any other Function's belief cycle. Its outputs drive the final reward signal and behavioral relevance scoring.
