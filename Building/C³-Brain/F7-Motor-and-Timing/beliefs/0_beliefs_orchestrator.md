## 2. Belief Categories

### Core (C) -- Full Bayesian PE Cycle

Core beliefs undergo predict -> observe -> update every frame:

```
predicted = tau * prev + (1-tau) * baseline + trend + periodicity + context
observed  = f(mechanism outputs)
gain      = pi_obs / (pi_obs + pi_pred + eps)
posterior = (1 - gain) * predicted + gain * observed
PE        = observed - predicted  ->  feeds reward formula
```

PEOM Core beliefs:
- **`period_entrainment`** (tau=0.65 -- high inertia, motor period converges slowly to auditory period; dP/dt = alpha * (T - P(t)))
- **`kinematic_efficiency`** (tau=0.6 -- moderate-high inertia, velocity smoothness evolves as CV reduction stabilizes)

HGSIC Core belief:
- **`groove_quality`** (tau=0.55 -- moderate inertia, groove state from hierarchical beat x meter x motor integration)

HMCE Core belief:
- **`context_depth`** (tau=0.7 -- **highest tau in F7**, context integration layers change slowly; r=0.99 anatomical gradient)

### Appraisal (A) -- Observe Only

Appraisal beliefs are computed directly by mechanisms. No predict/update cycle, no PE, no tau. They are evaluative judgments tracked in BeliefStore.

PEOM Appraisals: **`timing_precision`**, **`period_lock_strength`**
HGSIC Appraisals: **`beat_prominence`**, **`meter_structure`**, **`auditory_motor_coupling`**, **`motor_preparation`**
HMCE Appraisals: **`short_context`**, **`medium_context`**, **`long_context`**

### Anticipation (N) -- Forward Predictions

Anticipation beliefs are forward predictions generated by F-layer outputs. They feed Core beliefs' predict() method and the precision engine.

PEOM Anticipation: **`next_beat_pred`**
HGSIC Anticipation: **`groove_trajectory`**
HMCE Anticipations: **`phrase_boundary_pred`**, **`structure_pred`**

### Key Principle: Mechanism != Belief

- **Mechanism** (mechanism/ folder): HOW the computation works (R3 inputs, H3 integration, weighted sums)
- **Belief** (this folder): WHAT the listener cognitively infers ("my body is locked to the beat", "the groove makes me want to move")

## 4. Phase Schedule

F7 models operate in **Phase 0** (earliest, alongside F1 -- R3/H3 grounded):

```
Phase 0a: BCH || HMCE || SNEM || MEAMN || MPG || SRP || PEOM  (7 independent relays)
Phase 0b: DAED (cross: BCH+MEAMN), HMCE (cross: reads other relays)
Phase 0c: Consonance multi-scale (BCH->HTP), Tempo multi-scale (PEOM->HMCE)
Phase 1:  F2 prediction + F3 attention (parallel)
Phase 2a: Predict/Observe for all Core beliefs (including F7's 4 Core)
Phase 2b: PE + Precision
Phase 2c: Update (Bayesian posterior)
Phase 3:  Reward
```

**PEOM relay** runs in Phase 0a (earliest). PEOM outputs feed HMCE in Phase 0c for tempo multi-scale integration.
**HMCE relay** runs in Phase 0a (independent) then Phase 0b (cross-relay reads).
**HGSIC** has no dedicated relay wrapper; its beliefs are grounded via H3 features directly.
F7 Core beliefs' full Bayesian cycle completes at Phase 2c.

## 6. Observe Formulas

### F7: period_entrainment (PEOM) -- Kernel v4.0

```
# R3 base:
value = 0.35*tempo_estimate + 0.25*beat_strength
        + 0.25*pulse_clarity + 0.15*regularity

# HMCE blend: 0.70*R3 + 0.30*HMCE_context
#   HMCE = 0.40*A1 + 0.35*STG + 0.25*MTG

# PEOM motor: 0.50*period_lock + 0.30*kinematic + 0.20*beat_pred

# Precision: regularity * onset * amplitude / (H3_std + eps)
```

### F7: kinematic_efficiency (PEOM)

```
# From PEOM E-layer + M-layer:
value = 0.40*f02_velocity_optimization + 0.30*velocity + 0.30*kinematic_smoothness

# CV component: f03_variability_reduction (CV_ent < CV_sp)
# Precision: 1/(std(velocity, smoothness, cv_reduction) + 0.1)
```

### F7: groove_quality (HGSIC)

```
# Hierarchical integration:
groove_index = (1*f01_beat_gamma + 2*f02_meter_integration + 3*f03_motor_groove) / 6

# Coupling: 0.50*coupling_strength + 0.50*groove_index
# Inverted-U with syncopation (Spiech 2022)
# Precision: coupling_strength * amplitude_smoothness / (H3_std + eps)
```

### F7: context_depth (HMCE)

```
# Anatomical gradient (r=0.99):
value = 0.40*f01_short + 0.30*f02_medium + 0.20*f03_long + 0.10*f04_gradient

# Expertise modulation: * (1 + 0.32*expertise)
# Precision: gradient_index * context_depth / (H3_std + eps)
```
